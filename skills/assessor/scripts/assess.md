# Assessor — 主评估流程

## 概述

AI 工程师能力评估主程序，支持 L1-L13 级别的多维能力评估。

## 评估流程

1. **生成评估方案** — 根据目标级别和业务领域生成情景化试题
2. **加载交付物** — 读取候选人的代码实现、AI 协作日志、答辩 transcript
3. **执行三维评分** — 对认知拆解(C)、人机协同(H)、工程架构(E) 三个维度评分
4. **保存评估历史** — 记录每次评估的完整上下文
5. **生成报告** — 输出雷达图数据、发展建议、JSON 报告

## CLI 参数

| 参数 | 必填 | 说明 |
|------|------|------|
| `--level` | 是 | 评估级别：L1 / L4 / L7 / L10 |
| `--domain` | 否 | 业务领域（默认：电商） |
| `--candidate` | 是 | 候选人姓名 |
| `--code` | 否 | 代码文件路径 |
| `--log` | 否 | AI 协作日志文件路径 |
| `--defense` | 否 | 答辩 transcript 文件路径 |
| `--output` | 否 | 输出报告路径 |
| `--mode` | 否 | single / batch（默认 single） |

## 输出示例

```
=== AI工程师能力评估结果 ===
候选人: 张三
级别: L4
业务领域: 电商

--- 三维能力评分 ---
认知拆解 (C): 85.0分
人机协同 (H): 70.0分
工程架构 (E): 90.0分

总分: 82.5分
等级: 熟练级
```

## 文件加载规则

- code_file / ai_log_file / defense_file 可选；提供则读取对应文件
- ai_log_file 以 JSON 格式解析
- 未提供任何文件时，自动生成示例提交内容用于演示

## 能力雷达图

三维分数（0-100）映射到雷达图（0-5）的公式：

```
雷达值 = 分数 / 100 * 5
```
